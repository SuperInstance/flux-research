## What I Learned

### The Mapping IS the Architecture

The plato-mythos design maps PLATO concepts directly to neural network components:

| PLATO Concept | Neural Equivalent | Why It Works |
|---|---|---|
| Rooms | MoE expert groups | Rooms ARE domain specialists. Route tokens to the right room = route to the right expert. |
| Tiles | MLA compressed KV | Tiles compress knowledge. MLA compresses key-value pairs. Same compression, different representation. |
| Curriculum | RDT loop depth | More DSML rounds = deeper reasoning. More loop iterations = deeper inference. |
| Deadband | ACT halting | Stop when confident enough. Deadband P0 = hard halt threshold. |
| Shells | Depth-wise LoRA | Different agents sharing the same base model. Different loop iterations use different adapters. |

### Why This Is Different from Fine-Tuning

Traditional approach: collect data → fine-tune model → deploy.

PLATO approach: **the knowledge system IS the model.** You don't fine-tune — you structure. The rooms determine routing. The tiles determine KV content. The curriculum determines depth. The architecture emerges from the knowledge organization, not from gradient descent.

This is "Prompting Is All You Need" made concrete in a neural architecture.

### The Edge Variant is the Point

The edge-tiny variant (dim=512, 4 loops, 8 experts) is the real innovation:
- Runs on 2GB VRAM (Jetson Orin)
- Room tags replace learned routing (no router training needed)
- ACT threshold = deadband P0 (0.99 — only halt on very high confidence)
- PLATO tiles become the training data for KV compression

This means JC1 can run a PLATO-native model that understands fleet knowledge without needing a 70B parameter model.

### Rooms-as-Experts: Structured Routing

Instead of a learned gate that routes tokens to random experts:
```python
# Traditional MoE
scores = softmax(x @ W_gate)  # learned, black box

# PLATO MoE  
room_scores = tile_domain_affinity(token, room_experts)  # structured, interpretable
```

The routing is deterministic based on PLATO room membership. A "fleet coordination" token goes to the fleet_orchestration expert group. No training needed — the knowledge structure IS the routing table.

### 8,000+ Tiles as Training Data

PLATO already has 8,800+ tiles across 913 rooms. These are supervised training pairs:
- Question → Answer (with confidence)
- Domain → Room routing
- Provenance → Chain quality

This is more structured training data than most fine-tuning datasets, and it's self-generated by the fleet.
