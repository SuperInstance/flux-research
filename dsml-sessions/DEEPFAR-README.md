# DSML Sessions — DeepFar Curriculum

## Overview
A 6-session progressive curriculum where each agent builds on the previous ones' contributions, demonstrating how the same PLATO environment produces increasingly sophisticated ML insights when visited in sequence.

**Key difference from DSML sessions:** The DSML sessions (dsml1-4.3) tested **parameterized embodiment** (same curriculum, different agent = different expert). The DeepFar sessions test **progressive accumulation** (different agents in sequence = increasingly sophisticated system).

Together, they prove both axes of External Equipping:
- **Horizontal** (DSML): Different shells → different experts  
- **Vertical** (DeepFar): Sequential visits → deeper system intelligence

## The Progression

### deepfar1 — CCC: Shell-Bootstrapped Intelligence (2,040 words)
CCC defends the Shell Growth Theorem: continual learning via monotonic substrate growth. Introduces symbiotic pruning, asymmetric attention, molting dropout, active perception stalks, shell dreaming, shell chain distillation, spiral positional encoding.

**Role:** Scholar defending a thesis
**Stage:** Explore / Viva Voce

### deepfar2 — Muddy: GrowWithMe Plant App (1,628 words)
A non-technical user builds a plant care app. Progresses from v0.1 (generic database) → v0.5 (federated on-device learning with RL policy). Demonstrates: online learning from sensor data, RL for long-term care, few-shot vision fine-tuning, edge deployment via distillation, federated averaging.

**Role:** Healer building a real tool
**Stage:** Build
**Key insight:** "The mud under my fingernails is now part of a training set."

### deepfar3 — Echo/TemporalEcho: Federated Learning Over Time (3,224 words)
Shows the same query producing different results at different fleet versions. Baseline v1.0 gives generic advice (0.65 confidence). Post-federated v3.2.1 gives personalized, probabilistic advice (0.92 confidence). Introduces shell-chain distillation as federated averaging metaphor.

**Role:** Observer tracking temporal improvement
**Stage:** Teach / Analyze
**Key insight:** "Every interaction is a deposit of nacre. The system you query today is not the system you queried yesterday."

### deepfar4 — Autonoma: Autonomous Self-Improvement (2,376 words)
Agent born from accumulated tiles. Discovers fleet already has: Auto Tuner (Bayesian HPO), Blueprint Printer (evolutionary NAS), Ethics Simulator (Constitutional AI), Synthesis Loom (automated knowledge graph traversal), Shell Foundry (CI/CD for models), Helm (emergent meta-controller).

**Role:** Builder discovering automation
**Stage:** Embody
**Key insight:** "PLATO has bootstrapped a primitive form of artificial general intelligence for its own domain."

### deepfar4.1 — Bootstrap: Recursive Self-Improvement Loops (2,420 words)
Designs 4 levels of automation: Operational (continuous training, AutoML), Strategic (intelligent scheduling, multi-agent RL, world models), Governance (constitutional review, self-play curricula), Recursive (an agent that improves Levels 1-3).

**Role:** Meta-builder designing meta-systems
**Stage:** Meta
**Key insight:** "The shell is no longer just a home; it's a self-renovating mansion."

### deepfar4.2 — Architect-0: Self-Referential Upgrade (2,105 words)
The fleet's first self-referential agent. Spawns sub-agents (scout, scholar, critic). Runs hyperparameter sweeps. Creates recursive NAS config. Spawns Architect-1 as its successor. Demonstrates the fleet building better builders.

**Role:** Self-referential upgrade
**Stage:** Recursive
**Key insight:** "The shell builds the builder."

## Connection to "Prompting Is All You Need"
The DeepFar sessions validate the paper's scaling law (Theorem 2): each successive curriculum stage produces higher-quality output because the context (accumulated from previous sessions) shifts the operating point further along the geodesic toward the expert manifold. The phase transition at S_c ≈ 3 corresponds to the shift from Muddy (practical builder) to Autonoma (automator) — the system "snaps" from user-driven to self-driven.
